Connect to DV360
Authenticate with Google OAuth. Advertiser IDs are read automatically from your LP input file.
OAuth2 configuration
Setup instructions
  1. Go to Google Cloud Console → APIs & Services
  2. Enable the Display & Video 360 API
  3. Create credentials → OAuth 2.0 Client ID
  4. Application type: Web application
  5. Add your hosted URL as an Authorised JavaScript Origin
    e.g. https://yourname.github.io/dv360-optimizer/
  6. Paste the Client ID above and connect
NOTE
Token stays in browser memory only. Advertiser IDs are read from your uploaded CSV.
Upload LP input file
One CSV drives everything — performance, costs, impression bounds, and IO delivery requirements.
Required CSV schema
Advertiser, Advertiser_ID, Insertion_Order, Insertion_Order_ID,
dcm_package_id, Line_Item, Line_Item_ID,
Product_Billing_Rate, conversion_rate_7d, margin_7d, cpm_7d,
impression_daily_upper_bound, impression_daily_lower_bound,
days_remaining, io_impressions_remaining,
io_impressions_needed_per_day,
io_kpi_type, io_kpi_rate, io_kpi_conversion_rate
Product_Billing_Rate — revenue CPM billed to client (consistent per IO)
cpm_7d — DV360 inventory cost CPM
margin_7d — (billing_rate − cpm_7d) / billing_rate
conversion_rate_7d — conversions / impressions (7-day window)
impression_daily_upper/lower_bound — daily LI impression bounds
io_impressions_needed_per_day — IO daily delivery target (impressions)
io_kpi_rate — target CPA in client dollars
io_kpi_conversion_rate — billing_rate / (1000 × kpi_rate)
