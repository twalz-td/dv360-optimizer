Connect to DV360
Authenticate with Google OAuth. Advertiser IDs are read from your LP input file — no manual entry needed.
OAuth2 configuration
Setup instructions
  1. Go to Google Cloud Console → APIs & Services
  2. Enable the Display & Video 360 API
  3. Create credentials → OAuth 2.0 Client ID
  4. Application type: Web application
  5. Add your hosted URL as an Authorised JavaScript Origin
    e.g. https://yourname.github.io/dv360-optimizer/
  6. Paste the Client ID above and connect
NOTE
Your access token stays in browser memory only and is never transmitted elsewhere. Advertiser IDs are derived from your uploaded LP CSV.
Upload LP input file
One CSV drives everything — line item performance, costs, impression bounds, and IO delivery requirements. Advertiser IDs are read from this file to fetch live DV360 data automatically.
Required CSV schema
advertiser_id, line_item_id, io_id,
conversion_rate, cost_cpm, revenue_cpm,
imp_min, imp_max, io_required_impressions
advertiser_id — DV360 advertiser the LI belongs to
line_item_id — DV360 line item ID
io_id — Insertion order ID
conversion_rate — conversions ÷ impressions
cost_cpm — media cost per 1,000 impressions
revenue_cpm — revenue per 1,000 impressions
imp_min — minimum impressions for this LI
imp_max — maximum impressions for this LI
io_required_impressions — total IO must deliver (repeat same value per IO)
